# BiliHelper 产品功能文档

## 1. 产品概述

BiliHelper 是一个面向个人使用的 B 站视频内容解析、总结与问答工具。用户可以通过 Web 或 Android App 输入或分享 B 站视频链接，系统自动提取视频文案，生成摘要、章节划分，并支持历史记录、基于视频内容的问答和 Markdown 导出。

本产品聚焦于帮助用户快速理解 B 站视频内容，尤其适合课程、教程、知识分享、访谈、长视频和分 P 视频。

## 2. 用户角色

### 2.1 普通用户

普通用户可以：

- 注册和登录账号
- 配置自己的大模型 API 信息
- 提交 B 站视频链接进行分析
- 查看历史分析记录
- 查看视频文案、总结和章节
- 基于视频内容进行问答
- 导出 Markdown 文档

## 3. 平台范围

### 3.1 Web 端

Web 端提供完整功能，包括：

- 用户注册与登录
- API Key 配置
- 视频链接输入
- 视频分析任务创建
- 分析结果查看
- 历史记录管理
- 问答
- Markdown 导出

### 3.2 Android App 端

Android App 提供移动端使用能力，包括：

- 用户登录
- 手动输入 B 站视频链接
- 从 B 站 App 分享视频到 BiliHelper
- 查看分析进度
- 查看历史记录
- 查看总结、章节和文案
- 基础问答
- Markdown 导出或分享

## 4. 用户系统

### 4.1 注册

用户可以通过账号密码注册。

注册信息包括：

- 用户名或邮箱
- 密码
- 确认密码

注册要求：

- 用户名或邮箱不可重复
- 密码需要满足基础复杂度要求
- 注册成功后可直接登录或跳转登录页

### 4.2 登录

用户可以通过账号密码登录。

登录成功后，用户可以访问自己的：

- API 配置
- 视频分析历史
- 问答记录
- 导出内容

### 4.3 退出登录

用户可以主动退出当前账号。

退出后：

- 本地登录状态失效
- 需要重新登录后才能访问个人数据

## 5. 大模型配置

### 5.1 API 信息配置

用户可以配置自己的大模型 API 信息，用于总结、章节生成和问答。

配置项包括：

- 模型服务商
- API Key
- 模型名称
- 可选的 API Base URL
- 默认使用模型

### 5.2 配置管理

用户可以：

- 新增 API 配置
- 编辑 API 配置
- 删除 API 配置
- 设置默认配置
- 测试配置是否可用

### 5.3 使用规则

当用户发起视频分析或问答时，系统默认使用用户当前选中的模型配置。

如果用户未配置 API 信息，应提示用户先完成配置。

## 6. 视频链接提交

### 6.1 手动输入链接

用户可以在输入框中粘贴 B 站视频链接。

支持的链接类型包括：

- 普通 BV 链接
- AV 链接
- 移动端分享链接
- 带参数的 B 站网页链接
- 分 P 视频链接

### 6.2 Android 分享入口

Android App 支持从 B 站 App 接收分享内容。

用户在 B 站 App 中点击分享至 BiliHelper 后，BiliHelper 应自动识别分享文本中的视频链接，并进入分析确认页。

### 6.3 链接校验

系统需要校验：

- 是否为可识别的 B 站视频链接
- 是否已经分析过
- 是否为分 P 视频
- 是否需要用户选择分析范围

如果链接无效，应给出明确错误提示。

## 7. 视频信息解析

提交链接后，系统需要解析视频基础信息。

### 7.1 基础信息

需要展示：

- 视频标题
- UP 主名称
- 视频封面
- 视频链接
- 视频总时长
- 发布时间，如可获取
- 视频简介，如可获取

### 7.2 分 P 信息

如果视频包含多个分 P，需要展示：

- 分 P 总数
- 每个分 P 的标题
- 每个分 P 的时长
- 每个分 P 的序号
- 每个分 P 的分析状态

## 8. 分析范围选择

对于普通单 P 视频，用户可以直接开始分析。

对于分 P 视频，用户可以选择：

- 分析全部分 P
- 只分析当前 P
- 手动选择部分分 P

分析前需要展示预计处理范围，例如：

- 将分析 5 个分 P
- 已有 2 个分 P 存在历史结果
- 可选择跳过已分析内容或重新分析

## 9. 文案获取

### 9.1 字幕优先

系统应优先尝试获取 B 站已有字幕。

字幕来源包括：

- UP 主上传字幕
- B 站自动字幕
- B 站 AI 字幕，如可获取

### 9.2 ASR 兜底

如果视频没有可用字幕，系统应通过音频识别生成文案。

ASR 生成的文案应尽量包含：

- 文本内容
- 时间戳
- 分段信息

### 9.3 文案展示

用户可以查看完整文案。

文案展示要求：

- 按时间顺序排列
- 显示时间点
- 支持复制文本
- 支持按分 P 查看
- 支持折叠或展开长文案

## 10. 视频总结

### 10.1 单 P 总结

每个已分析的分 P 都应生成独立总结。

总结内容包括：

- 核心主题
- 内容概述
- 关键观点
- 重要结论
- 适合快速阅读的短摘要

### 10.2 全视频总结

对于分 P 视频，系统应支持生成全视频级别总结。

全视频总结包括：

- 整体主题
- 各分 P 内容概览
- 跨分 P 的知识结构
- 重点内容汇总
- 适合复习的结论整理

### 10.3 重新生成总结

用户可以对已有分析结果重新生成总结。

重新生成时可以选择：

- 只重新生成当前 P
- 重新生成全部分 P
- 只重新生成全视频总结

## 11. 章节划分

### 11.1 章节生成

系统需要根据文案自动生成章节。

每个章节包括：

- 开始时间
- 结束时间，如可生成
- 章节标题
- 章节描述
- 关键内容摘要

### 11.2 章节展示

章节应按时间顺序展示。

用户可以通过章节快速理解视频结构。

### 11.3 时间点交互

章节中的时间点应可点击。

点击后应：

- 在 Web 端跳转到对应 B 站视频时间点，或定位到对应文案位置
- 在 Android 端尝试打开 B 站 App 或浏览器对应时间点
- 同时高亮对应章节或文案片段

## 12. 问答功能

### 12.1 基础问答

用户可以基于视频内容提问。

系统回答时应参考：

- 视频文案
- 视频总结
- 章节信息
- 分 P 信息

### 12.2 问答范围

用户可以选择问答范围：

- 当前分 P
- 全部已分析分 P
- 指定若干分 P

### 12.3 回答内容

回答应尽量包含：

- 直接回答
- 依据说明
- 相关时间点
- 相关分 P 信息

### 12.4 问答历史

系统应保存每个视频下的问答记录。

用户重新打开历史视频时，可以继续查看之前的问题和回答。

## 13. 历史记录

### 13.1 历史列表

用户可以查看自己分析过的视频列表。

列表展示：

- 视频标题
- 封面
- UP 主
- 分析时间
- 分析状态
- 是否为分 P 视频
- 已分析分 P 数量

### 13.2 历史详情

用户点击历史记录后，可以查看：

- 视频基础信息
- 分 P 列表
- 文案
- 总结
- 章节
- 问答记录
- 导出入口

### 13.3 历史搜索与筛选

历史记录应支持：

- 按标题搜索
- 按 UP 主搜索
- 按分析时间排序
- 按分析状态筛选

### 13.4 删除历史

用户可以删除历史记录。

删除时应提示用户确认。

## 14. Markdown 导出

### 14.1 单 P 导出

用户可以导出单个分 P 的 Markdown 文档。

内容包括：

- 视频标题
- 分 P 标题
- 原始链接
- 总结
- 章节
- 文案

### 14.2 全视频导出

对于分 P 视频，用户可以导出完整 Markdown 文档。

内容包括：

- 视频基础信息
- 全视频总结
- 分 P 目录
- 每个分 P 的总结
- 每个分 P 的章节
- 每个分 P 的文案

### 14.3 导出内容选项

用户可以选择导出内容：

- 仅总结
- 总结 + 章节
- 总结 + 章节 + 文案
- 包含问答记录

### 14.4 文件命名

导出的 Markdown 文件应使用易识别名称，例如：

- `视频标题.md`
- `视频标题-P1-分P标题.md`

## 15. 任务状态

视频分析过程可能较长，系统需要展示任务状态。

状态包括：

- 等待中
- 正在解析视频
- 正在获取字幕
- 正在进行语音识别
- 正在生成总结
- 正在生成章节
- 已完成
- 已失败

失败时应展示失败原因，并支持重新尝试。

## 16. 异常与提示

系统需要处理以下异常情况：

- 链接无效
- 视频不存在
- 视频不可访问
- 分 P 信息获取失败
- 字幕获取失败
- ASR 失败
- 大模型调用失败
- 用户 API Key 无效
- 分析任务超时
- Markdown 导出失败

所有异常都应提供清晰提示，并尽量给出下一步操作建议。

## 17. 非目标功能

当前阶段不包含以下功能：

- 视频文件下载
- 大规模商用能力
- 多用户团队协作
- 公开视频分享社区
- 自动订阅 UP 主更新
- 视频内容二次分发
- 高级权限系统
- 支付系统

## 18. MVP 功能范围

第一版建议实现：

- 用户注册登录
- API Key 配置
- Web 端视频链接提交
- 普通视频与分 P 视频识别
- 字幕获取
- 无字幕时 ASR 兜底
- 文案展示
- 单 P 总结
- 单 P 章节
- 历史记录
- Markdown 导出
- 基础问答

Android 端第一版建议实现：

- 登录
- 手动输入链接
- 接收 B 站分享
- 查看分析结果
- 查看历史记录
